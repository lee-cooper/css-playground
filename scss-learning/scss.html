Preprocessor

Sass (or SCSS) provides additional features that are not present in vanilla CSS, such as nesting, mixins , inheritance, and other feature that make writing robust and scalable CSS easier. It accomplishes this by compiling sass / scss files into regular css.

The pre-processor can be run using the npx sass command with a input file(s) and an output destination file specified.

For example, I started out using npx sass --watch scss-learning/styles.scss main.css, which with the --watch flag listened for any changes in the input files and outputted the file to a main.css file. Problem is, it put this at the root of the project, rather than in this file folder.

I then learned that you can watch file folders for any changes in the input files, and also specify an output directory to compile the scss styles to.

I ended up using this command to do this: npx sass --watch scss-learning/styles.scss:scss-learning/dist/main.css
